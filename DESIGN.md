# poke_fantasy — 게임 기획 · 상세 사양 (정본)

이 문서는 현재 게임의 기획 내용과 세부 동작을 한곳에 정리한 **정본(source of truth)**이다.
시간순 패치 이력은 `BALANCE.md`를 참고한다.

> 규칙 원칙: 플레이어에게만 적용되는 규칙은 상대 AI에게도 항상 대칭으로 적용한다.

---

## 1. 개요

- 장르: 그리드 전술 RPG(무기 종류로 전직이 결정되는 weapon-kind grid tactics).
- 맵: 10×10 타일. 팀당 최대 4명. A팀=플레이어(화면 아래), B팀=AI(화면 위)로 **상하 대결**.
- 화면 흐름: **홈(팀 선택)** → 전투 시작(바로 전투) / 인벤토리(별도 세팅) → **전투** → **결과**.

## 2. 화면 · UI

- **홈(팀 선택)**: A/B팀 캐릭터 선택. "⚔ 전투 시작"으로 바로 전투. "🎒 인벤토리"로 세팅 화면.
- **인벤토리**: 캐릭터별 능력치 확인·**레벨업 포인트 분배**(남은 포인트가 있으면 각 능력치 옆 `+` 버튼) · 무기/방어구/보조장비 장착 · **기술 로드아웃(최대 4개)** 설정. 로스터 인스턴스를 직접 편집하고, 전투 시작 시 고유 id로 복제해 사용(양 팀 중복 선택 대비).
- **전투 화면(세로 구성: 상단 바 — 맵 — 하단 바로 화면 꽉 채움)**
  - **상단 바**: `🕑 시간대 · 날씨` + **행동 순서 바**(살아있는 전원을 스피드순 정렬, 현재 차례 ▶·금색 강조, 암흑(시야 밖) 상대는 `???`).
  - **맵(카메라 뷰포트)**: 타일 86px 고정, 보드가 화면보다 크며 **카메라가 현재 유닛을 따라 스크롤**(스크롤바 숨김, 숨은 상대 차례엔 이동 안 함). 스타크래프트식 시점.
  - **하단 행동 일자바(약 2배 두께)**: 좌측 상단 = 현재 캐릭터 정보 + 전투 로그 한 줄, **우측 상단 = 버프/디버프 칩(+남은 턴)**(버프 녹색·디버프 적색). 아래에 **기술 2×2 그리드** + 우측(무기교체 / 취소·확인).
  - **아군 정보 카드**: 맵에서 아군(현재 차례가 아닌 아군 포함)을 누르면 좌측 상단에 정보 카드 표시(원화·이름·Lv·무기/방패·HP·능력치·이동/시야·버프/디버프). 닫기 버튼 또는 행동 시 닫힘.
- **표현 디테일**
  - 캐릭터는 직업·성별별 PNG 원화로 표시(아군 후면 back, 적 정면 front). 스프라이트 못 찾으면 아무것도 안 그림.
  - 체력바: 아군은 잔량별 초록/노랑/빨강, **적군(B팀)은 항상 빨간색**.
  - 타일 아이콘: 숲 🌲 / 언덕 ⛰️ / 물 🌊 / 바위 🪨 / 불 🔥(평지는 색만).
  - 안개(암흑, 스타크래프트식): **한 번도 못 본 타일**은 균일한 어두운 회색(지형 종류 알 수 없음). **한 번 밝혔던(탐사) 타일**은 시야를 벗어나도 **지형 색을 어둡게 기억**해서 표시(현재 유닛 위치는 안 보임). **현재 보이는 타일**은 지형 전체 색. 날씨·시간대 틴트는 스크롤 안 되는 무대에 올려 균일하게 덮음.
  - 공격 모션: 시전자가 상대 방향으로 돌진(아군 위·적 아래), 피격 대상은 흔들림 + 붉은 섬광.

## 3. 거리 체계 (중요)

- **시야 · 이동 · 사거리 · 기술 범위/반경 = 마름모(맨해튼 거리)**. 대각선 1칸 = 거리 2.
- 예외: **보호(가드) 반경**은 마름모 1칸(직교 인접 4칸).
- 스킬 명중 판정의 정렬/근접성 등 AI 내부 계산도 맨해튼 기준.

## 4. 캐릭터 · 스탯

- **레벨업 포인트 분배제**: 점수로 찍을 수 있는 능력치 5종은 전원 초기값 5에서 시작하고, 레벨업마다 3포인트를 인벤토리 화면에서 자유 분배한다(`unspentStatPoints` → `spendStatPoint`). 최대 레벨 100.
  - `hp`(체력): 0이 되면 전투 불능(사망 처리).
  - `attack`(근력): 물리 공격력 산정 + **적재량**에 영향.
  - `magicAttack`(지력): 마법 공격력 산정 + **정신력**에 영향.
  - `speed`(스피드): 행동 순서 + **회피율**에 영향.
  - `endurance`(지구력): **이동력** 산정에 영향.
- **점수로 찍을 수 없는(전부 파생) 능력치**:
  - **적재량**(kg): 기본 5kg + 근력 5당 1kg(내림). 장착·소지 중인 모든 무기·방어구 무게 합이 이를 넘을 수 없다.
  - **이동력**: 아래 5장 참고(최대 5칸).
  - **정신력**: `(지력 / 305) × 0.7` — 상대가 가한 상태이상·부가효과를 무시할 확률(최대 70%). `mentalResistChance`.
  - **회피율**: `(스피드 / 305) / 2 + (자신 레벨 − 공격자 레벨) / 100`, [0,1] 클램프 — 공격을 피해 없이 회피할 확률. `evasionChance`. 명중 판정을 통과한 공격에 대해 방어측이 한 번 더 굴린다(단, 활의 '집중' 발동 시 생략).
  - **방어력**: 방어구 + 방패로만 얻는다(기본 스탯 아님). 데미지 공식에서 그대로 차감.
  - **사거리**: 무기 종류별 고정.
  - **시야**: 기본 5 고정(보정치는 6장 참고). 시야 밖 타일의 상대 행동·정체는 확인 불가.
- `weaponMastery`(무기별 전직 티어 0~6), `skillLoadout`(전투에 들고 갈 스킬 최대 4).
- 로스터(10명, 전원 레벨 10 · sight 5): 환도무사·대검전사(검), 철퇴호위·대곤파괴자(둔기), 각궁사수·장궁저격수(활), 화염술사·대지술사(지팡이), 사제·현자(마법서). 5개 능력치 합은 전원 5×5+27=52(레벨10 기준 분배 포인트 27).

## 5. 이동

- **이동력 = (지구력 / 30) + 1**, 최대 5칸. 5를 넘는 초과분(a, "5+a"로 표기)은 실사용 칸 수에 반영되지 않는 버퍼로 남아 이동 감소 페널티를 **우선 상쇄**하고, 그래도 남는 페널티만 실제 이동력(≤5)에서 차감한다(END가 높을수록 이동 디버프에 강함).
- **이동력 1 미만이면 2턴에 1칸만 이동**(라운드가 홀수일 때만 1칸, `moveStepsForRound`).
- 소수점은 내림(floor)해서 칸 수로 사용. **마름모(직교 4방향 BFS)**: 대각선은 직교 2칸을 거치므로 비용 2.
- 지형: **물·언덕은 진입 가능하지만 넘어서 이동 불가**(경로가 그 칸에서 끝남). **바위는 진입 불가**(장애물). 점유 타일 진입 불가.
- 페널티/보너스: **물 위 -1**(급류 시 +1로 상쇄), 다리부상(legHit) magnitude, 날씨(비=중장 -0.5, 눈=경장 -0.5).

## 6. 시야 · 전장의 안개

- `effectiveSight` = 기본 시야 + 보정, 최소 1.
  - 천리안(farSight) +1, **언덕 위 +1**, **밤 -2**, **비/눈 -1**.
- 전장의 안개(스타크래프트식 3단계): 현재 시야=지형 전체 색 / 탐사 완료(과거에 봤음)=지형 색을 어둡게 기억 / 미탐사=균일한 암흑(지형 종류 알 수 없음). 탐사 완료 구역엔 현재 유닛 위치는 표시되지 않음.
- **숲 은폐**: 숲 안 유닛은 (투시 없으면) 맨해튼 1칸 밖에서는 안 보임.
- **불 조명**: 불타는 타일 포함 주변 1칸의 캐릭터/타일은 시야와 무관하게 확인 가능(양 진영).

## 7. 전투 흐름

- 턴 순서: `effectiveSpeed`(스피드×각성보정 + 무기 baseSpeed) 내림차순, 동률은 rng.
- **데미지 공식**(`calculateDamage`, 상세는 `damage.ts`):
  - `최종공격력 = (주스탯/6 + 무기공격력) × 숙련도~100% 랜덤값 × 기술위력` (주스탯은 물리 스킬=근력, 마법 스킬=지력, 마법부여 중엔 근력+지력 합산).
  - `최종데미지 = 최종공격력 − 방어력` (방어구+방패, 창의 관통 발동 시 0), 최소 1로 floor.
  - **숙련도 랜덤값**: 전직 티어(0~6)에 따른 하한(placeholder: 티어0=50% ~ 티어6=100%)과 100% 사이 균등 난수.
  - 명중 판정(스킬 자체 정확도) 통과 후, **방어측이 회피율만큼 회피 판정**을 한 번 더 굴린다(회피 성공 시 무피해, 활의 '집중' 발동 시 생략). 그 다음 무기 부가효과(관통/급소 등) 반영.
- **속성 상성**(불<물<나무<강철<땅<불): 약점 1.3배, 강점 0.7배, 무속성은 영향 없음.
- **기술 로드아웃**: 실제 전투에선 로드아웃(∩ 사용 가능)만 사용. 하나도 없으면 **기본 공격 주먹**(공통/물리/위력 50%/명중 100/사거리 1/무제한, 선택 목록엔 미노출)으로 대체.
- **즉시 발동 UX**: 타겟 필요 없는 기술(자신/자신범위/아군범위)은 버튼만 눌러도 발동. 대상 지정형은 **타겟 클릭 즉시 발동**(확인 불필요). 대기 중 이동이 있으면 이동 후 발동. 확인 버튼은 이동/무기교체/대기(턴 종료)용.
- **무기 교체**: 하단바에서 순환 선택 후 확인. 숙련 티어<3이면 교체가 그 턴을 소모(엔진 규칙).
- 처치·경험치: 막타를 넣은 캐릭터가 경험치 획득, 레벨업 시 전직 포인트 지급.
- **AI 행동 로직**(대칭 원칙에 따라 플레이어도 같은 시야/정보 제약을 받음, `pickAiAction`):
  1. 도발 대상이 있으면 최우선으로 그 대상을 추적.
  2. 없으면 현재 시야 안의 가장 가까운 적을 추적, 사거리 안이면 공격 스킬 사용.
  3. **시야 안에 적이 하나도 없어도 가만히 있지 않는다**: 각 진영은 상대 유닛을 마지막으로 확인한 위치를 `GridBattle.knownEnemyPositions`에 기억해두고(시야를 벗어나도 유지), 그 위치 중 가장 가까운 곳을 향해 계속 이동한다. 한 번도 목격한 적이 없으면(교전 전) 맵 중앙을 향해 정찰하듯 전진한다.
  4. 예상 위치는 **이동에만** 쓰이고, 공격은 실제로 시야 안에 있거나 도발로 강제된 대상에게만 시도한다(추측만으로 공격하지 않음).

## 8. 장비(무기 · 방어구) · 전직

- **장비 종류**: 무기, 방어구(방패는 무기 종류 중 하나로 별도 취급, 아래 참고).
- **무기 종류**(kind, 9종): 검(sword)·둔기(blunt)·창(spear)·활(bow)·석궁(crossbow)·단검(dagger)·투척(thrown)·지팡이(staff)·마법서(tome). ※ 창/석궁/단검/투척은 전용 스킬 미구현(공통 스킬 + 폴백 주먹만 사용 가능).
  - **근접**: 검·둔기·창·단검. **원거리·마법**: 활·석궁·투척·지팡이·마법서.
  - 방패(shield)는 별도의 무기 종류로, 스킬 없는 순수 스탯(방어력 보너스) 아이템이며 한손 무기와 병용한다(양손 무기 장착 시 자동 해제). 방어구(아래)와는 독립된 슬롯.
- **무기는 10레벨 단위 등급**(Lv.10~100)으로 존재하며, 공격력은 종류에 관계없이 `착용 레벨 / 2`로 동일하다(단, **단검은 3/4 배율**). 무기 무게는 테스트 단계라 전 종류 1kg로 고정.
- 무기 템플릿(사거리·속도·손잡이만 종류별로 고정, 발췌): 환도/대검(검 r1), 철퇴/대곤(둔기 r1), 창(창 r2), 각궁/장궁(활 r2), 석궁(석궁 r2), 주술서/마법서(마법서 r1), 법장/스태프(지팡이 r2), 단검(단검 r1), 투척무기(투척 r2), 원형/타워 방패(방어 +15/+25).
- **방어구 종류**(ArmorKind, 4종): 천(cloth)·가죽(leather)·중갑(mail)·판금(plate). 무기·방패와 별개의 3번째 장비 슬롯(`equippedArmorId`).
  - 방어구도 10레벨 단위 등급. 방어력 = 같은 레벨 무기의 기준공격력(단검 제외, `착용 레벨 / 2`) × 종류별 배수(천 ×0.7 / 가죽 ×0.9 / 중갑 ×1.1 / 판금 ×1.3).
  - 방어구 무게: 천 0.5kg / 가죽 1kg / 중갑 3kg / 판금 5kg.
- **착용 레벨 제한**: 무기·방어구·방패 모두 `캐릭터 레벨 ≥ 장비 레벨`이어야 장착 가능(인벤토리에서 미달 시 버튼 비활성화 + 필요 레벨 표시).
- **인벤토리 슬롯**: 기본 장착 장비(무기+방패+방어구) 외에 적재량 범위 내에서 최대 2개까지 추가로 소지할 수 있다.
- **전투 중 장비 교체**: 하단바의 장비교체 버튼이 무기·방어구를 하나의 순환 목록으로 묶어 보여준다(레벨 미달 항목 제외). 무기 교체는 숙련 티어<3이면 그 턴을 소모하고, 방어구 교체는 항상 그 턴을 소모한다.
- 전직: 5레벨 단위, 최대 티어 6. `getUsableSkillIds`가 티어별 해금 스킬 반환.
  - 티어1 보너스: 검 위력+10%, 둔기 명중+5, 활 명중+10, 지팡이/마법서 위력+10%.
  - 티어5 패시브(설계 placeholder): 검=저체력 위력↑, 둔기=방어↑, 활=시야+1, 지팡이=속성강점 페널티 무시, 마법서=사용횟수+1.
- **무기 부가효과**(`weaponEffects.ts`, 공격 적중 시 30% 확률로 발동, `rollWeaponProc`):
  - 검=**출혈**(2턴간 매 턴 최대체력 1/8 피해) · 둔기=**기절**(2턴 동안 매 턴 별도로 30% 확률 행동불가 판정) · 창=**관통**(방어구+방패 방어력 전부 무시) · 활=**집중**(회피율 무시) · 석궁=**급소**(데미지 ×1.5) · 방패=없음.
  - 마법서·투척무기는 종류가 고정되어 있지 않고 **인스턴스마다 위 5개 효과 중 하나를 선택**해 지닌다(`WeaponInstance.procEffect`, 로스터 예시: 사제=급소, 현자=집중).
  - 출혈·기절은 상대의 **정신력** 저항 판정을 통과해야 실제로 적용된다(다른 상태이상 감소 효과와 동일한 저항 규칙).
- **단검 이도류**: 주무기가 단검일 때만 보조 슬롯(방패 슬롯과 동일 슬롯 재사용)에 단검을 하나 더 장착할 수 있다(`equipOffhandDagger`). 다른 무기·방패와는 병용 불가하며, 주무기를 단검이 아닌 것으로 바꾸면 보조 단검도 자동 해제된다. 추가타 등 별도 보너스는 없음(순수 장비 슬롯 허용).

## 9. 스킬 (요약)

- **공통**: 주먹(폴백), 강타(물리 100%), 주술(마법 100%, 지팡이 속성), 보호(주변 1칸 아군 대신 피격 2턴), 도발, 낙석(언덕 전용), 마법부여-불/물/나무/강철/땅, 급류, 등반, 천리안, 투시.

- **둔기**: 다리타격(T2 이동↓), 단결(T4 아군 방어↑), 분쇄(T6 강타).

## 10. 지형 · 타일

1. **평지**: 특수 효과 없음(색: 연두).
2. **숲**: 인접 1칸 밖에서 은폐(내부 유닛 안 보임). **숲 안에서는 근접 무기만 행동 가능**. 진입/통과 가능. 불타면 평지로 변함.
3. **물**: 진입 가능하나 **넘어서 이동 불가**. **물 위 이동력 -1**(급류로 상쇄).
4. **언덕**: **등반 버프 없이도 진입 가능**, 넘어서 이동 불가, **언덕 위 시야 +1**, **올라선 턴엔 행동 불가**(등반 버프 시 예외).
5. **바위**: 장애물(진입 불가). **원거리·마법 공격이 바위를 넘어 타격 불가**(직선 경로 차단).
6. **불(타일 상태)**: 해당 타일 캐릭터 매 턴 최대체력 1/4 피해. 주변 1칸 조명(시야 무관 확인). 지속 후 소멸.

## 11. 날씨 · 시간대

- **날씨**(전투 시작 시 랜덤): 맑음 / 비 / 눈 / 폭염.
  - 비: 중장(검·둔기·창·방패) 이동 -0.5, 시야 -1.
  - 눈: 경장(원거리·마법) 이동 -0.5, 시야 -1.
  - 폭염: 턴 시작 시 최대체력 1/16 피해(정신력=마법공격력 기반, 최대 70% 확률로 저항).
- **시간대**(랜덤): 아침 / 낮 / 저녁 / 밤. **밤 시야 -2**. 보드에 옅은 색 틴트.

## 12. 상태이상 (StatusEffectType)

- 버프: guarding(보호), elementEnchant(마법부여), riverSurge(급류), climbing(등반), farSight(천리안), forestVision(투시), swordAwaken(각성), bluntUnity(단결), bowCrit(급소), focused(집중).
- 디버프: taunted(도발), legHit(다리부상), bleeding(출혈, 검 부가효과), stunned(기절, 둔기 부가효과).
- 각 상태는 남은 턴 수를 가지며 턴 시작에 감소. 하단바 우측 상단 칩으로 표시.
- 상대가 거는 디버프·부가효과(도발/다리부상/약화/출혈/기절)는 대상의 **정신력** 확률만큼 무시된다.
