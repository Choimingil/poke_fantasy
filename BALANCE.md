# 밸런스 · 패치 로그

> 원칙: 플레이어에게만 적용되는 규칙은 상대 AI에게도 자동으로 동일하게 적용한다.
> (시야/안개, 날씨 이동 페널티, 폭염 피해 등은 양 진영 대칭)

이 문서는 그리드 전술 RPG(weapon-kind grid tactics) 엔진 기준의 밸런스·표현 계층 변경 이력을 기록한다.

---

## 보드 카메라 시점(스타크래프트식) · 하단 행동 바

- 타일 하나를 정사각형(86px)으로 고정. 보드가 화면(뷰포트)보다 커지고, 카메라가 현재 행동 유닛을 화면 중앙으로 따라가며 스크롤(스크롤바 숨김). 숨은 상대 차례엔 카메라를 옮기지 않음.
- 화면 세로 구성: 상단 바(시간/날씨/행동순서) — 맵(카메라) — 하단 행동 일자바. 맵 높이는 상·하단 바를 제외한 나머지를 전부 차지.
- 행동 컨트롤(기술/취소·확인·턴넘기기)을 좌측 하단 플로팅 패널에서 맵 하단 전체 폭 일자바로 이동. 기술 버튼은 한 줄에서 가로 스크롤.

---

## 공격 모션 · 기본 공격(주먹) · 플로팅 기술 패널

- **공격 모션**: 공격 시 시전자가 상대 방향으로 돌진(아군=위, 적=아래)하고, 피격 대상은 흔들림 + 붉은 섬광. 플레이어·AI 모두 적용.
- **기본 공격 주먹**: `(스킬 없음)` 버튼 제거. 사용 가능한 스킬이 하나도 없으면 자동으로 주먹으로 대체.
  - 주먹: 공통 / 물리 / 위력 50% / 명중 100 / 사거리 1 / 사용제한 없음. 선택 목록·기본 로드아웃에는 노출되지 않는 폴백 전용.
- **플로팅 기술 패널**: 기술/확인 버튼을 맵 좌측 하단 플로팅 패널로 이동(⚔ 버튼으로 접기/펼치기).
- **정사각형 프레임**: 전투 보드를 남은 공간에서 최대 정사각형으로 잡고 가운데 정렬(카메라 스크롤 유지).

---

## 행동 순서 바 · 상태 텍스트 위치

- **상단에 행동 순서 바 추가**: 참여 중인 모든 캐릭터를 스피드(무기 보정 포함) 내림차순으로 나열하고, 현재 차례를 ▶·금색 테두리로 강조.
  - 암흑(플레이어 시야 밖)으로 공개되지 않은 상대는 `???`로 표시(수·행동 순서 위치는 노출되나 정체는 숨김).
- **상태/설명 텍스트 이동**: 현재 유닛 정보와 최신 로그 텍스트를 기술 버튼 바로 위(액션 패널)로 이동.

---

## 이동 미리보기 · 시간대 · 타일/안개 표현

- **이동 미리보기**: 아군이 이동할 칸을 지정해 행동을 고르는 대기 상태에서, 캐릭터 토큰을 예정 칸으로 미리 옮겨 보여준다(카메라도 예정 칸을 추적).
  - 이때 시야 계산은 아직 이동 전 위치 기준이라, 예정 칸 인접 1칸의 숲에 숨은 상대는 **행동을 확정한 이후에** 드러난다.
- **시간대**: 전투 시작 시 아침/낮/저녁/밤 중 무작위 결정. 맵 상단에 시간대·날씨 표시. 밤/저녁/아침에는 보드에 옅은 색 틴트(현재는 정보성/연출 위주).
- **평지 타일 색**: 갈색 → 연두색 계열로 변경.
- **암흑(안개) 표현**: 완전 검정이 아니라 투명도를 준 검정으로 덮어 시야 밖 지형이 희미하게 비치도록 완화.

## 스킬 로드아웃 · 화면 구성 · 흐름 개편

- **스킬 로드아웃(최대 4개)**: 전투에 들고 갈 스킬을 캐릭터별로 최대 4개까지 선택.
  - `skillLoadout`을 캐릭터에 추가. `getLoadoutSkillIds`가 사용 가능 스킬 ∩ 로드아웃(최대 4)을 반환하며,
    플레이어 UI와 AI가 모두 이 목록만 사용해 대칭 적용.
  - 미설정 시 사용 가능 스킬의 앞 4개(강타·주술·보호·도발 등)를 기본 사용.
- **타일 아이콘**: 숲🌲 / 언덕⛰️ / 물🌊 / 화염🔥 아이콘 표시(평지는 아이콘 없음). 숲은 실루엣이 항상 보임.
- **상하 배치**: 좌우 대결에서 상하 대결로 변경. 아군(A팀)은 화면 아래, 적(B팀)은 위. 스폰/지형을 가로 방향으로 재배치.
- **이동 가능 칸 강조**: 파란 오버레이 + 펄스 링으로 이동 가능한 칸을 뚜렷하게 표시(공격 대상은 붉은 오버레이).
- **흐름 개편**: 홈에서 캐릭터 선택 후 바로 전투 시작. 능력치·기술·무기 세팅은 별도 "인벤토리" 버튼에서 진행.
  - 인벤토리는 로스터 인스턴스를 직접 편집하고, 전투 시작 시 고유 id로 복제해 사용(양 팀 중복 선택 대비).

## 표현 계층 복원 + 날씨/지형 재이식 (element-grid-tactics 재작성 이후)

`feature/element-grid-tactics` 재작성으로 기술/전직/속성 시스템은 새 아키텍처로 재구현되었으나,
이전 버전에서 반영했던 표현 계층과 날씨 시스템이 사라져 아래를 다시 반영했다.

### 표현 계층

- **캐릭터별 원화 스프라이트 복원**: 전투 유닛이 단일 플레이스홀더(SVG)로 덮이던 것을 되돌리고,
  직업·성별별 PNG 원화(`assets/jobs/{spriteJob}-{gender}-{front|back}.png`)로 표시.
  - 각 로스터 캐릭터에 `spriteJob`/`gender` 매핑 추가.
  - 아군(A팀)은 후면(back), 상대(B팀)는 정면(front) 방향으로 표시.
  - 스프라이트를 찾지 못하면 아무것도 렌더하지 않아 캐릭터가 사각형 이미지로 덮이지 않는다.
- **카메라 추적 복원**: 전투 보드를 고정 픽셀(64px) 셀로 만들고 스크롤 뷰포트에 담아,
  현재 행동 유닛을 화면 중앙으로 부드럽게 추적. 단, 플레이어에게 보이지 않는(안개/숲 은폐) 상대의
  차례에는 카메라를 옮기지 않고 아군 시점을 유지한다.
- **전장의 안개 / 숲 은폐 복원**:
  - 플레이어(A팀) 시야 밖 타일은 안개로 덮는다. 숲 타일은 실루엣이 항상 보이되 내부 유닛은 은폐.
  - 플레이어 시야에 들어오지 않은 상대 유닛 토큰은 표시하지 않는다.
  - 시야 밖 상대는 타겟팅(공격 대상 지정)도 불가.

### 날씨 (맑음 / 비 / 눈 / 폭염)

- 전투 시작 시 무작위로 하나 결정(양 진영 공통).
- **비**: 중장(근접: 검·둔기·창·방패) 계열 `rawMove` −0.5.
- **눈**: 경장(원거리·마법: 활·석궁·단검·투척·지팡이·마법서) 계열 `rawMove` −0.5.
- **폭염**: 턴 시작 시 최대 체력 1/16 피해. 단, 정신력(마법공격력 기반, 최대 70%) 확률로 저항.
  - 저항 확률 = min(0.7, 마법공격력 / 40).
- **맑음**: 페널티 없음.

이동 페널티는 `effectiveMove`에, 폭염 피해는 턴 시작 처리에 통합되어 플레이어·AI 모두에 대칭 적용된다.
